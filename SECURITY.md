# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this repository, please report it responsibly. **Do not** open a public GitHub issue for security vulnerabilities.

### How to Report

1. **Go to the Security tab** of this repository
2. Click **"Report a vulnerability"** → **"Start a draft security advisory"**
3. Provide details about the vulnerability:
   - Description of the vulnerability
   - Steps to reproduce (if applicable)
   - Potential impact
   - Suggested fix (if you have one)

Or email security details to the repository maintainer if available.

## Security Advisories

Once a vulnerability is confirmed, we will:
1. Create a security advisory
2. Work on a fix in a private fork
3. Request a CVE (if applicable)
4. Publish the advisory after a fix is available
5. Credit the reporter (unless they prefer anonymity)

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| Current | ✅ Yes            |
| Previous | ✅ Yes            |
| Older   | ❌ No             |

Please keep your local version updated with the latest main branch commits.

## Security Features Enabled

This repository has the following security features enabled:

- ✅ **Secret Scanning** - Detects accidentally committed secrets
- ✅ **Dependabot Alerts** - Monitors for vulnerable dependencies
- ✅ **Code Scanning** - Identifies vulnerabilities in code
- ✅ **Private Vulnerability Reporting** - Allows private disclosure

## Questions?

If you have questions about security, please contact the repository maintainer through:
- GitHub Discussions
- Private security advisory
- The contact information in the repository README
