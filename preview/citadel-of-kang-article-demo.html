This demo intentionally uses neutral article text. Chambers of AK remains the first implementation, but Citadel of Kang must be reusable across different websites, routes and content types.

1. Module Boundary

Citadel of Kang is the theme pack. Citadel Manager is the future admin-only website management product. The public website should only load safe frontend modules and generated public configuration.

The theme pack should not contain private admin credentials, repository write tokens, CMS backend logic, client portal data or Chambers-specific private implementation details.

2. Preserve Existing Features

Existing Chambers/Citadel features remain the baseline. The current dark mode toggle, mobile navigation behavior, social link strategy, article index decisions, SEO discipline and PowerShell 7 patch workflow must be preserved unless expressly approved for replacement.

3. Route-Agnostic Design

Citadel modules must not depend only on routes such as /updates/*.html. They should activate through data attributes, public-safe config and fallback markup detection.

4. Article Experience

The Article Index module should support long-form pages, articles, guides, insights and documentation. Reading time and reading progress should be optional standalone modules that no-op safely when markup is absent.

5. Accessibility Foundation

Citadel should include skip links, semantic landmarks, keyboard-safe navigation, visible focus states, reduced-motion handling, readable dark/light contrast and accessible module behavior.

6. Adaptation Without Replication

Allowed

Study mature systems to identify missing capabilities and convert them into original Citadel-designed features.

Not Allowed

Do not copy proprietary code, assets, UI layouts, commercial builder logic or naming conventions.

7. CMS Separation

Citadel Manager / Website CMS should later handle content editing, theme switching, template management, plugin/module management, SEO settings and publishing. It should not be merged into the theme pack.

8. Client Portal Separation

The future client portal for matter status, documents and hearing dates must be a separate secure product with authentication, authorization, private storage and audit logs.

9. Validation Checks

Before rollout, run JavaScript syntax checks, git whitespace checks and manual desktop/mobile/dark/light/keyboard tests. Production rollout should happen only through a controlled PowerShell 7-compatible patch.
